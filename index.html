02 — How it works

Request flow, step by step

What actually happens between you typing the URL and seeing this page.

01
Browser sends a DNS query
Your browser asks Route 53 to resolve the domain name. Route 53 returns the CloudFront distribution's address, not S3 directly — this is what keeps S3 private.
Route 53 → A record (Alias)
02
CloudFront checks its cache
The request hits the nearest CloudFront edge location (there are 400+ worldwide). If the file is already cached there, it's returned immediately — no trip to S3 needed. This is why the site loads fast globally.
CloudFront edge → Cache hit or miss
03
On cache miss — fetch from S3
If the file isn't cached, CloudFront fetches it from the S3 origin bucket over a private AWS backbone link, caches it, and serves it. The S3 bucket itself stays private — only CloudFront can access it via an Origin Access Control (OAC) policy.
CloudFront → S3 origin (private OAC)
04
HTTPS enforced end-to-end
CloudFront terminates SSL using a free certificate from AWS Certificate Manager (ACM). HTTP requests are automatically redirected to HTTPS. S3 itself doesn't handle SSL — that's entirely CloudFront's job.
ACM certificate · HTTP → HTTPS redirect
03 — Services used

AWS services in this project

Three services, each doing exactly one job well.

Amazon S3
Object storage that holds all static files — HTML, CSS, JS, images. Acts as the origin for CloudFront. The bucket is kept private; no public access is enabled on S3 itself.
Free: 5 GB storage · 20,000 GET requests/month
Amazon CloudFront
Content delivery network with 400+ edge locations globally. Handles SSL termination, HTTP→HTTPS redirect, caching, and serving files close to the user for low latency.
Free: 1 TB data transfer · 10M requests/month
Route 53
AWS managed DNS. Routes your custom domain to the CloudFront distribution using an Alias record. Also supports health checks and failover routing for more advanced setups.
$0.50/hosted zone/month (optional — skip to stay fully free)
AWS Certificate Manager
Provisions and auto-renews free SSL/TLS certificates for your domain. Must be created in us-east-1 region to work with CloudFront, regardless of where your other resources are.
Free: public SSL certificates are always free

05 — Bucket Policy

S3 bucket policy used

This policy allows only CloudFront (via OAC) to read from the bucket. The bucket itself stays private.

bucket-policy.json 
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "AllowCloudFrontOAC",
      "Effect": "Allow",
      "Principal": {
        "Service": "cloudfront.amazonaws.com"
      },
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::YOUR-BUCKET-NAME/*",
      "Condition": {
        "StringEquals": {
          "AWS:SourceArn": "arn:aws:cloudfront::ACCOUNT-ID:distribution/DIST-ID"
        }
      }
    }
  ]
}
